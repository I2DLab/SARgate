Prediction

The Prediction tab provides a QSAR-oriented workflow for training a machine-learning model on the current dataset and using it to estimate activities for molecules with missing labels or for newly added external compounds.

Input Options

The manager bar allows the user to select:

  • Training scope, choosing whether the model should be trained on the prepared dataset, the full dataset, or a specific subset.
  • Activity type, defining the endpoint to be learned by the model.
  • Feature set, including Morgan fingerprints alone or fingerprints combined with RDKit descriptors.
  • Model, selecting a specific regression algorithm or an automatic model-selection mode based on cross-validation.
  • Read undefined, enabling inclusion of activity annotations expressed with qualifiers such as <, <=, >, or >=.

Prediction Output

After the model is trained, SARgate displays the prediction output as a table and accompanying visual panels. Rows corresponding to compounds with known activities report both real and predicted values, allowing direct comparison between observed and estimated responses. Compounds lacking experimental labels are still predicted and can therefore be prioritised for follow-up analysis.

Model-quality metrics are calculated from the same cross-validated predictions used during training, providing a compact view of the reliability of the current model configuration. When automatic model selection is used, SARgate evaluates multiple candidate estimators and keeps the best-performing one for the chosen scope and activity endpoint.

External Compounds

Once a model has been fitted, the Prediction tab can also be used to score external molecules. These additional compounds are merged into the visible prediction output so they can be compared directly against the original library while preserving the trained model and the current activity scale.
