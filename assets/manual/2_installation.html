Installation

SARgate was developed and tested on Python 3.13.
It supports Python 3.13 and any newer Python release.
Earlier Python versions may work but are not officially validated.

SARgate is distributed both as source code and as operating-system-specific executables. The steps below describe how to prepare a Python environment and install dependencies when running SARgate from source code. These steps are required only when running SARgate from source code, and are not required when running the executable build of SARgate.

If you are using a SARgate executable, no Python or dependency setup is required and you can launch the app directly.

Download & Install Python

macOS:

Install using the official Python installer, available at the Python website, or using a package manager such as Homebrew: 

brew install python@3.13

Linux (Ubuntu / Debian):

sudo apt install python3

Windows:

Install Python from Microsoft Store, or the official Python installer available at the Python website. During installation, enable the checkbox "Add to PATH" (important for CMD and PowerShell compatibility)

Choose an Installation Method

SARgate can be installed from source using either a standard Python virtual environment created with venv or a Conda environment.

Both methods use the project requirements.txt file to install dependencies. Choose the one that best matches your current Python workflow.

Install with venv + pip

Install / Enable pip

This step is only required for the venv workflow. In many Python installations, pip is already available.

macOS / Linux:

python3 -m ensurepip --upgrade

Windows (CMD or PowerShell):

python -m ensurepip --upgrade

Check pip version

macOS / Linux:

python3 -m pip --version
which pip

Windows CMD:

python -m pip --version
where pip

Windows PowerShell:

python -m pip --version
Get-Command pip

Create the Virtual Environment

macOS / Linux:

python3 -m venv SARgate-venv

Windows CMD / PowerShell:

python -m venv SARgate-venv

Activate the Virtual Environment

macOS / Linux:

source SARgate-venv/bin/activate

Windows CMD:

SARgate-venv\Scripts\activate

Windows PowerShell:

SARgate-venv\Scripts\Activate.ps1

Note for Windows

If script execution is blocked (error: "script execution is disabled"):

1. Open Windows PowerShell as Administrator

2. Check current execution policy: 

Get-ExecutionPolicy

3. If the result is `Restricted`, update it: 

Set-ExecutionPolicy RemoteSigned

4. Confirm the change by pressing: 

Y

Install Required Packages

macOS / Linux:

python3 -m pip install -r requirements.txt

Windows CMD / PowerShell:

python -m pip install -r requirements.txt

Install with Conda

If you prefer Conda, create and activate a dedicated environment first, then install SARgate dependencies using the same requirements.txt file.

Create the Conda Environment

All OS:

conda create -n SARgate-env python=3.13

Activate the Conda Environment

All OS:

conda activate SARgate-env

Install Required Packages

Run this command after activating the Conda environment:

python -m pip install -r requirements.txt

Important

If rdkit or another scientific dependency fails to install via pip inside Conda, install the missing package from conda-forge and then rerun the requirements command: 

conda install -c conda-forge rdkit
python -m pip install -r requirements.txt

Verify Installed Packages

macOS / Linux with venv:

python3 -m pip list

Windows or Conda:

python -m pip list

Run SARgate

Before launching SARgate, make sure your chosen environment is active.

macOS / Linux:

python3 launcher.py

Windows CMD / PowerShell:

python launcher.py
